# stock_system_good_v2 — 전시장 패널 기반 급등 예측

기존 `stock_system`(v1)의 근본 한계를 해결하는 재설계.

## v1의 문제

v1은 거래대금 상위 1,500 프리필터 → 점수 통과 후보(하루 ~83종목)만
`score_history.json`에 기록해 학습했다. 즉 "**내가 이미 고른 후보 중
누가 급등하나**"를 배울 뿐, "**전체 시장에서 누가 급등하나**"는 볼 수 없었다.
v1 자체 진단으로도 급등주 97.9%가 프리필터에서 탈락 — 내일 급등할 종목은
오늘 조용해서 후보에 없는 경우가 대부분이다. 28거래일 × 83종목(양성 288건)
으로는 다중 조건 패턴의 통계 검증도 불가능했다.

## v2의 접근

매일 쌓이길 기다리지 않고 **전 종목(~2,400 보통주) × 과거 3년을 한 번에
역추적**한다. 약 170만 행, 양성 수만 건의 패널에서 GBM으로 학습하고,
운용 지표(매일 상위 k픽의 적중률 = precision@k)로 워크포워드 검증한다.

- 라벨: `max(High[t+1..t+k]) / Close[t] - 1 ≥ 10%` (k=1, 3, 5)
  — 종가가 아닌 **고가** 기준: 장중 10% 터치 후 밀린 케이스 포함
- 피처: 거래량(잠복→회복, 회전율), 가격 구조(52주 신고가 거리, 박스 압축,
  이평 배열, 골든크로스 간격), 변동성/캔들(NR7, ATR 확장, RSI, 갭),
  시장 국면(지수 수익률, 상승 종목 비율, 어제 급등 종목 수),
  섹터 전염(같은 섹터 어제 급등 수), **횡단면 백분위**(당일 전 종목 대비 순위)
- 누수 차단: 모든 피처는 t 시점까지의 데이터만 사용, 워크포워드 학습 꼬리에
  10일 엠바고 (라벨 윈도우가 검증 기간과 겹치는 경계 누수 차단)

## 사용법

```bash
python3 build_panel.py all        # 수집(~10분, 재개 가능) + 패널 생성
python3 train_panel.py eval 5     # 워크포워드 검증 (D+5, precision@k/lift)
python3 train_panel.py patterns 5 # 피처별 십분위 lift 표 (패턴 리포트)
python3 train_panel.py train 5    # 최종 모델 저장 → panel_model_5d.pkl
```

## 1단계 검증 결과 (2026-07-06, 최근 12개월 워크포워드)

| 라벨 | 기저율 | AUC | p@5 | lift | 나이브 베이스라인* |
|---|---|---|---|---|---|
| `j1` 다음날 하루 종가 +10% | 2.0% | 0.80 | **26.5%** | **x14.6** | 12.0% (x6.1) |
| `5` 5일 내 고가 +10% 터치 | 19.9% | 0.72 | **80.6%** | **x4.3** | 53.7% (x2.7) |

\* 나이브 = 모델 없이 당일 volat20 상위 5픽. 모델이 2배 이상 상회 →
변동성 선택 효과를 넘는 진짜 신호 존재.

### 발견된 패턴 (십분위 lift, `pattern_lift_label_jump_1d.json`)

- **활동성이 곧 신호**: 회전율 상위 10% = 3.8x, 변동성 상위 = 3.7x,
  거래량비 상위 = 2.5x. 내일의 급등은 "이미 시동 걸린" 종목에서 나온다.
- **v1의 '박스권 압축' 가점은 역방향**: 박스 압축 상위(가장 조용한) 10%
  = **0.08x** — 초박스권은 급등 확률이 기저의 1/12. 압축 자체는 강한
  음의 신호이고, v1이 +20점을 주던 로직과 정반대.
- **잠복(quiet_streak)도 역신호**(0.73x)지만 **잠복→회복 전환
  (vol_recovery)은 1.75x** — v1의 '전환이 핵심' 학습과 일치.
- NR7 = 0.69x (v1 학습과 일치: 단독으론 역방향)
- 수익률은 U자형: 급락주(ret5 하위 10% = 2.0x)와 급등주(상위 = 3.1x)
  양끝이 높고 중간이 낮다 — 반등과 모멘텀 두 갈래.
- 전염 확인: 어제 시장 급등 종목 수 상위 = 1.9x, 같은 업종 어제 급등 = 1.7x,
  MA 정배열 = 1.6x.

## 2단계: 실매매 시뮬레이션 결과 (2026-07-07)

**핵심 발견 — 예측력 ≠ 수익.** j1 top5를 다음날 시가에 사면 평균 **-3.2%/건**:
평균 시가 갭 **+7.7%**가 예측 수익을 선반영. 급등 예측이 맞을수록 시초가가
높게 열려 못 먹는다.

**검증된 매매 규칙**: 전체 top5 랭킹 중 **픽 당일 아직 +15% 미만인
종목만** D+1 시가 매수 → 5거래일 보유 (거래비용 0.25% 차감):

| 랭킹 모델 | 건/일 | 평균/건 | 승률 | 마이너스월 | 성격 |
|---|---|---|---|---|---|
| o5 (시가 진입 라벨) | ~2.0 | **+1.04%** | 45% | 5/12 | 안정형 |
| j1 (다음날 급등 라벨) | ~0.9 | **+2.17%** | 39% | 6/12 | 고분산 (대박 소수가 견인) |

- 중앙값은 음수 (o5 -2.0%) — **소수의 큰 승자가 평균을 만든다**. 손실 월을
  버티는 자금 관리 필수. 월 수익 범위 o5 [-5.6%, +7.6%] / j1 [-19%, +32%].
- '비상한가 풀에서 top5 채우기'(픽 단계 필터)는 신호 희석으로 실패(-0.5%) —
  반드시 **전체 랭킹 top5 → 비상한가만 골라 매매** 순서여야 한다.
- 수급(외인/기관, 네이버 트렌드 API) 피처는 j1 기준 기여 없음
  (p@5 26.5→24.7%, 노이즈 범위) — 가격/거래량 신호에 이미 흡수된 듯.

## 운용 (자동화 완료)

**LaunchAgent `com.stocksystem.v2_predict`가 평일 18:00 자동 실행**
(`run_nightly.sh`): 수급 갱신 → 가격 갱신·패널·스코어링·v1 정밀검증 →
리포트(`reports/picks_YYYY-MM-DD.md`) → 성과 추적(`reports/performance.md`).
기존 v1 작업들(19:00 재학습, 20:00 학습, 20:10 스캔)과 시간 분리.

```bash
bash setup_launchd_v2.sh          # 등록 (이미 완료) / remove로 삭제
python3 predict_today.py          # 수동 실행
python3 track_performance.py      # 성과만 재계산
tail -f logs/nightly_$(date +%Y%m%d).log
```

리포트의 `✅ 매매` 표시(top5 & 당일 +15% 미만)만 시가 매수, 5일 보유.
나머지 상위 픽은 관찰용. 각 픽은 `picks_history.json`에 누적돼
track_performance가 실현 수익·적중률을 매일 사후 검증한다.

**v1 2단 결합**: 상위 10픽은 v1(stock_system)의 `analyze_one`으로 정밀
재검증(v1_bridge.py, 별도 프로세스) — 세력 점수·ML 티어·진입 타이밍이
리포트에 병기된다. v2 = 후보 생성기(전 종목 랭킹), v1 = 정밀 검증기.

**KRX 로그인**: `krx_config.json`(권한 600, **git 미추적**)에 자격증명 보관,
`config_v2.py`가 KRX_ID/KRX_PW 환경변수로 주입. **2026-07-08 로그인 정상
(CD001)** — 전종목 일자별 OHLCV, 투자자별 수급(금액), **공매도 잔고** 등
KRX 전용 API 사용 가능. 현 파이프라인은 네이버 소스로 충분해 그대로 두고,
공매도 잔고는 향후 피처 후보 (수급 피처가 무기여였던 전례상 기대는 보수적으로).
